The VictoriaMetrics write api can be accessed via port 8428 on the following DNS name from within your cluster:
    vm-victoria-metrics-single-server.monitoring.svc.cluster.local.

Metrics Ingestion:
  Get the Victoria Metrics service URL by running these commands in the same shell:
    export POD_NAME=$(kubectl get pods --namespace monitoring -l "app=" -o jsonpath="{.items[0].metadata.name}")
    kubectl --namespace monitoring port-forward $POD_NAME 8428

  Write URL inside the kubernetes cluster:
    http://vm-victoria-metrics-single-server.monitoring.svc.cluster.local.:8428/<protocol-specific-write-endpoint>

  All supported write endpoints can be found at https://docs.victoriametrics.com/victoriametrics/single-server-victoriametrics/#how-to-import-time-series-data

  E.g: for Prometheus:
    http://vm-victoria-metrics-single-server.monitoring.svc.cluster.local.:8428/api/v1/write

Read Data:
